## Technical Implementation Notes

### State Management
- Use React Context or Zustand for global state (league settings, player values, draft state)
- Persist to localStorage so users don't lose their draft progress on refresh
- Consider IndexedDB for larger projection datasets

### CSV Parsing
- Use PapaParse for robust CSV handling
- Handle edge cases: quoted commas, special characters in names, various encodings

### Calculations
- Memoize expensive calculations (inflation adjustments affect all players)
- Use Web Workers for initial value generation if dataset is large
- Debounce inflation recalculations during rapid draft entry

### Data Validation
- Validate CSV structure before accepting
- Warn on missing required columns
- Handle duplicate player names gracefully

### Responsive Design
- Desktop-first (this is a draft day tool, likely on a laptop)
- But ensure tablet usability for those following along on iPad
- Mobile: at minimum, show inflation-adjusted values table
